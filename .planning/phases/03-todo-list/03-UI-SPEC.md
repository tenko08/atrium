## Design System

| Property | Value |
|----------|-------|
| Tool | none |

## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | CSS Equivalent | Usage |
|-------|-------|---------------|-------|
| xs | 4px | `0.25rem` | Icon gaps within rows (SyncDot right margin), input border-radius |
| sm | 8px | `0.5rem` | Inline element spacing (course name margins, button padding vertical) |
| md | 16px | `1rem` | Button padding horizontal, form field spacing, standard margins |
| lg | 24px | `1.5rem` | Section spacing below SyncButton, heading bottom margin |
| xl | 32px | `2rem` | Page padding on all sides |
| 2xl | 48px | `3rem` | Not used in this phase |
| 3xl | 64px | `4rem` | Not used in this phase |

Exceptions:
- List item row bottom margin: 12px (`0.75rem`) — existing pattern from Phase 2, continue for consistency
- Sub-element spacing (last synced line, error line below button): 6px (`0.375rem`) — existing SyncButton pattern, continue
- Touch target minimum for checkbox + icon buttons: 24px (matches existing row height at `0.75rem` margin cadence)

**Source:** Extracted from existing codebase (index.tsx, SyncButton.tsx, MissingCredentials.tsx). All values map to multiples of 4px.

## Typography

| Role | Size | CSS Value | Weight | Line Height | Usage |
|------|------|-----------|--------|-------------|-------|
| Body | 16px | `1rem` | 400 (regular) | 1.5 | Task titles, form labels, button text, group headers |
| Label | 14px | `0.875rem` | 400 (regular) | 1.5 | Due date text, duration display, helper text, last synced timestamp, form input placeholder text |
| Micro | 13px | `0.8rem` | 400 (regular) | 1.4 | Last synced line below SyncButton (existing pattern — `0.8rem`) |
| Heading | 24px | `1.5rem` | 700 (bold) | 1.2 | Page `<h1>` ("Atrium"), section group headers (`<h2>`) for course names and date buckets |

Font weight constraint: use only 400 (regular) and 700 (bold). No intermediate weights (500, 600) — monospace system fonts do not reliably support them.

**Source:** Extracted from index.tsx (`1rem`, `0.875rem`, `0.8rem`), MissingCredentials.tsx (`1.5rem` heading). One additional role (Micro) retained from existing SyncButton pattern.

## Color

| Role | Hex | Usage |
|------|-----|-------|
| Dominant (60%) | `#ffffff` | Page background, row backgrounds, form input backgrounds |
| Secondary (30%) | `#f4f4f4` | Code block surfaces (MissingCredentials), inline form background when active, disabled input background |
| Muted text | `#666666` | Course name labels, due date auxiliary text, last-synced timestamp, placeholder text |
| Subtle text | `#999999` | Due date text in list rows (less prominent than course names) |
| Border | `#cccccc` | Button borders, input borders, group separator lines |
| Destructive | `red` (CSS keyword) | Inline error messages, failed sync error text |

Accent palette (10% — reserved-for list):

| Accent | Hex | Reserved for |
|--------|-----|-------------|
| Sync new (green) | `#22c55e` | SyncDot when `syncStatus === 'new'` — this element only |
| Sync updated (orange) | `#f97316` | SyncDot when `syncStatus === 'updated'` — this element only |
| Completion strikethrough (muted) | `#999999` | Completed task title text + strikethrough decoration — this element only |

Accent is NOT used for: general interactive states, hover states, focus outlines, toggle buttons, or form borders. Use border `#cccccc` and background `#f4f4f4` for interactive feedback instead.

**Completion state:** `text-decoration: line-through; color: #999999` on the task title `<strong>` element. No background color change on the row.

**Hover state (manual task row):** No background color change on the row. Pencil and trash icon buttons appear (`visibility: visible`) on `onMouseEnter` via React state. Icons use `color: #666666` at rest, no further hover treatment needed.

**Grouping toggle active state:** Active button uses `background: #f4f4f4; border: 1px solid #cccccc`. Inactive button uses `background: transparent; border: 1px solid transparent`. No color accent on toggle.

**Source:** All values extracted from existing codebase. No new colors introduced.

## Component Inventory

Components the executor must build for this phase:

### 1. GroupingToggle
- Renders two adjacent buttons: "By Course" and "By Due Date"
- Active button: `background: #f4f4f4; border: 1px solid #cccccc; font-family: monospace; font-size: 1rem; padding: 0.25rem 0.75rem; border-radius: 4px`
- Inactive button: same dimensions, `background: transparent; border: 1px solid transparent`
- Placed directly below `<h1>` and above the SyncButton (or after SyncButton — positioned by Claude's discretion)
- Recommendation (Claude's discretion): place toggle above SyncButton, flush left, `marginBottom: '1rem'`

### 2. AssignmentGroup (section wrapper)
- `<section>` element containing a `<h2>` group label and a `<ul>` of task rows
- Group `<h2>`: `font-size: 1rem; font-weight: 700; margin-bottom: 0.5rem; margin-top: 1.5rem; font-family: monospace; color: #333`
- First group has no top margin (`margin-top: 0`)
- `<ul>`: `list-style: none; padding: 0; margin: 0`

### 3. TaskRow (Canvas assignment row)
- `<li>` with `margin-bottom: 0.75rem; display: flex; align-items: baseline`
- Checkbox: `<input type="checkbox">` with `margin-right: 0.5rem`
- Title: `<strong>` element. If `completed`: add `text-decoration: line-through; color: #999999`
- Course name (course view only): `<span style={{ color: '#666', margin-left: '0.5rem' }}>(courseName)</span>`
- Due date: `<span style={{ color: '#999', margin-left: '0.5rem', font-size: '0.875rem' }}>— due {formatDueDate(dueAt)}</span>`
- SyncDot: existing component, rendered before title
- No edit/delete icons — Canvas rows are read-only (D-14)

### 4. ManualTaskRow (manual task row)
- Same base as TaskRow with two additions:
  - `onMouseEnter` / `onMouseLeave` handlers set `hoveredId` state
  - When `hoveredId === task.id`: render pencil button (&#9998;) and trash button (&#128465;) at right end of row
  - Icon buttons: `background: none; border: none; cursor: pointer; font-size: 1rem; margin-left: 0.25rem; color: #666; font-family: monospace`
  - Row uses `display: flex; align-items: baseline; justify-content: space-between`
  - No SyncDot (manual tasks have no syncStatus)

### 5. InlineEditRow (replaces ManualTaskRow during edit)
- Triggered by pencil icon click
- Renders structured inputs in place of the display row (same layout as creation form)
- Save: Enter key on title input OR checkmark button (✓)
- Cancel: Escape key OR ✕ button
- Submit calls `PATCH /assignments/:id`, updates local state, exits edit mode

### 6. ManualTaskCreationForm (bottom of Manual section)
- Always visible at the bottom of the "Manual" group — not in a modal (D-08)
- Form `<div>` with `margin-top: 1rem; padding-top: 1rem; border-top: 1px solid #cccccc`
- Fields (all inline with labels):
  - **Name**: `<input type="text" placeholder="Task name" required>` — full width, `width: '100%'`
  - **Duration**: `[__] h [__] m` — two `<input type="number" min="0">` with `width: '3rem'` each, literal " h " and " m " text between
  - **Due date (optional)**: `[DD] / [MM] / [YYYY]` plus optional `[HH] : [MM]` — five `<input type="number">` elements with separators; labeled "(optional)"
- Submit button: `+ Add Task` — `font-family: monospace; font-size: 1rem; padding: 0.5rem 1rem; border: 1px solid #ccc; background: #fff; border-radius: 4px; cursor: pointer`
- Validation error (inline, below affected field): `font-size: 0.875rem; color: red; margin-top: 0.25rem`

### 7. EmptyGroup (shown when a group has no items)
- Not rendered — if a group has no items after filtering, the entire `<section>` is omitted (simpler than showing empty state per group)
- Exception: the "Manual" section always renders even when empty (to show the creation form)

## Interaction Contracts

### Completion Toggle (D-05, D-06, D-07)
1. User clicks checkbox on any task row
2. Optimistic: immediately apply strikethrough + move row to bottom of its group in local state
3. API call: `PATCH /assignments/:id/complete` with `{ completed: true/false }`
4. On error: revert local state, show inline error (see Copywriting Contract)
5. No animation on movement (keep it simple — Claude's discretion per CONTEXT.md)

### Hover-Reveal Edit/Delete (D-11, D-12, D-13)
1. `onMouseEnter` on a manual task `<li>`: set `hoveredId = task.id`
2. `onMouseLeave`: set `hoveredId = null`
3. Pencil (&#9998;) click: set `editingId = task.id` — renders InlineEditRow
4. Trash (&#128465;) click: call `DELETE /assignments/:id`, remove from local state, call `setHoveredId(null)`
5. No confirmation dialog for delete (D-13)

### Inline Edit (D-12)
1. InlineEditRow pre-fills with current title, estimatedMin (converted to hours + minutes), dueAt (converted to day/month/year + time)
2. Enter or ✓: submit `PATCH /assignments/:id`, update local state, set `editingId = null`
3. Escape or ✕: set `editingId = null` — no state change, no API call

### Grouping Toggle (D-03, D-04)
1. Toggle between "By Course" and "By Due Date" updates `groupBy` state
2. No API call — pure client-side re-grouping of existing `assignments` array
3. State does not persist across page loads (session-only)

### Manual Task Creation (D-08, D-09, D-10)
1. Form always visible at bottom of Manual section
2. Submit: validate name (required) and duration (required, minimum 1 minute total)
3. Due date: optional. If day/month/year filled but no time, default to 23:59 local time. Validate date coherence (reconstructed date must match inputs — guard JS date overflow)
4. On success: prepend task to local assignments state, clear form fields
5. On error: show inline error below affected field

## Interaction States

| State | Visual Treatment |
|-------|-----------------|
| Task incomplete | Title at weight 400, `color: inherit`, no decoration |
| Task complete | `text-decoration: line-through; color: #999999` on title |
| Row hovered (manual task) | Pencil + trash icons become visible (`visibility: visible`) |
| Row not hovered | Pencil + trash icons hidden (`visibility: hidden`) — preserve row height |
| Edit mode active | Row replaced with InlineEditRow; pencil/trash not shown |
| Toggle button active | `background: #f4f4f4; border: 1px solid #cccccc` |
| Toggle button inactive | `background: transparent; border: 1px solid transparent` |
| Button disabled (syncing) | `opacity: 0.7; cursor: not-allowed` (existing SyncButton pattern) |
| Form input focus | Browser default focus ring — do not suppress |
| Number input (hours/minutes/date) | `width: 3rem; text-align: center` — constrained width makes structure clear |

**Visibility vs display for hover icons:** Use `visibility: hidden` (not `display: none`) so the row height does not shift when icons appear. This prevents the list from jumping on hover.

```
[h1] Atrium
[GroupingToggle] [By Course] [By Due Date]
[SyncButton]

[AssignmentGroup — course name OR date bucket]
  [TaskRow / ManualTaskRow] x N
  [InlineEditRow] (when editing)
  [ManualTaskCreationForm] ← Manual section only

[AssignmentGroup — next group]
  ...
```

- Page wrapper: `padding: '2rem'; font-family: 'monospace'` (existing pattern)
- Manual section position (Claude's discretion): rendered last, after all Canvas course groups in "By Course" view. In "By Due Date" view, manual tasks with a due date appear in their correct bucket; manual tasks without a due date appear in the "No due date" bucket.
- Max width: none — full width of page container (existing pattern)
