The module exports one public function `runSync()` and several internal helpers (exported for testing):

```typescript
export function extractNextLink(header: string | null): string | null
export function resolveDueAt(assignment: CanvasAssignment, userSectionIds: Set<string>): string | null
export function computeSyncStatus(
  canvasId: string,
  title: string,
  dueAt: number | null,
  existingMap: Map<string, { title: string; dueAt: number | null }>
): 'new' | 'updated' | 'unchanged'
export async function runSync(): Promise<{ assignmentCount: number }>
```

Implementation details for `runSync()`:
1. Read `CANVAS_BASE_URL` from `process.env`, strip trailing slash: `process.env.CANVAS_BASE_URL!.replace(/\/+$/, '')`
2. Read `CANVAS_API_TOKEN` from `process.env`
3. Fetch active courses: `GET /api/v1/courses?enrollment_type=student&enrollment_state=active&per_page=100`
4. For each course, fetch assignments: `GET /api/v1/courses/{id}/assignments?include[]=overrides&per_page=100`
5. For each course, fetch user's enrolled sections: `GET /api/v1/courses/{id}/enrollments?user_id=self&per_page=100` — collect `course_section_id` values into a `Set<string>`
6. For each assignment, call `resolveDueAt(assignment, userSectionIds)` to get correct due date
7. Before upserting, query existing canvas assignments: `SELECT canvas_id, title, due_at FROM assignments WHERE source = 'canvas'`
8. For each assignment, call `computeSyncStatus()` to determine new/updated/unchanged
9. Reset all canvas assignments to `'unchanged'` first, then upsert incoming assignments with computed status
10. Upsert using Drizzle `onConflictDoUpdate` targeting `canvasId`

Rate limit handling: If fetch returns 429, sleep 1 second (`await Bun.sleep(1000)`) and retry the same request once. If second attempt also 429, throw error.
